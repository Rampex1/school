For every Weekly Task List, create a .md file which is a large table
It should include the following columns:
✓ Sprint (indicating which sprint the task is planned to be
done)
✓ TaskID (should be a unique identifier accross the tasks)
✓ StoryID (the backlog item ID associated with the task)
✓ Task description
✓ Assigned team member (name)
✓ Estimated Effort (in hours)
✓ Status (Todo / In progress / Done)
✓ Actual Effort (in hours; for Done tasks)
✓ Start Date
✓ End Date
Some points to be noted:
o Should be able to identify the backlog items associated with
each task
o Task description should be clear and terse
o Each team member (including Proxy Product Owner, except
for the Scrum Master) should to at least: (invent from ECSE428 - Group 3 - Team Info.pdf)
• 3 hours of task per week
• Two different types of tasks (ex: programming vs
testing)
o Task States:
o Todo
• Should not not be assigned to anyone
• Should not include a start to end date
• Should include estimated effort
o In progress
• Should be assigned to someone
• Should include a start date but not an end date
• Should include estimated effort but not actual
effort
o Done
• Should be assigned to someone
• Should include a start and an end date
• Should include estimated effort and actual effort
o Should indicate what each unit refers to in the effort section
(in case each unit does not indicate one hour
